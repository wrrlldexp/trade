# MoneyBot v2 — Серверные требования

## Минимальные требования

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| **CPU** | 2 vCPU | 4 vCPU |
| **RAM** | 4 GB | 8 GB |
| **Диск** | 20 GB SSD | 50 GB SSD |
| **ОС** | Ubuntu 22.04+ / Debian 12+ | Ubuntu 24.04 LTS |
| **Docker** | 24.0+ | Последняя стабильная |
| **Сеть** | 10 Mbit/s | 100 Mbit/s |
| **Пинг до Binance** | < 200 мс | < 50 мс |

## Потребление ресурсов по сервисам

Данные из `docker-compose.prod.yml` (лимиты):

| Сервис | CPU | RAM | Примечание |
|--------|-----|-----|-----------|
| **PostgreSQL** | 1.0 | 512 MB | Основная БД, растёт с количеством данных |
| **Redis** | 0.5 | 128 MB | Pub/sub + heartbeat, потребление минимальное |
| **Backend (FastAPI)** | 1.0 | 512 MB | 2 uvicorn workers в prod |
| **Worker** | 1.0 | 512 MB | Основная нагрузка — тики сеток |
| **Frontend** | 0.5 | 256 MB | Статика, нагрузка минимальная |
| **Nginx** | 0.25 | 64 MB | Reverse proxy |
| **Итого** | **4.25** | **~2 GB** | Суммарный лимит |

> Реальное потребление обычно 50-70% от лимитов.

## Расчёт нагрузки по сеткам

### Worker (основная нагрузка)

Каждая активная сетка:
- **CPU**: ~0.05-0.1 vCPU при тике каждую секунду
- **RAM**: ~10-20 MB (зависит от количества ордеров)
- **Сеть**: ~8 weight на тик к Binance API

| Количество сеток | CPU worker | RAM worker | Binance weight/мин |
|-----------------|------------|------------|---------------------|
| 1-3 | 0.3 vCPU | 150 MB | ~480 |
| 5-10 | 0.5-1.0 vCPU | 200-400 MB | ~800-1200 |
| 10-15 | 1.0-1.5 vCPU | 400-600 MB | 1200 (лимит!) |
| 15+ | Требуется масштабирование | | Нужно несколько API-ключей |

### Binance API лимиты

| Лимит | Значение | Что означает |
|-------|----------|-------------|
| Request weight | 1200/мин | Общий лимит на все запросы |
| Order rate | 10 ордеров/сек | Лимит на создание ордеров |
| Raw requests | 6100 за 5 мин | Количество запросов |

**Один тик сетки потребляет ~8 weight**: get_ticker (1) + проверка ордеров (2-5) + возможные place/cancel (1-2).

**Безопасный максимум: 10-15 сеток на один API-ключ.**

### Bybit API лимиты

| Лимит | Значение |
|-------|----------|
| Request rate | 120 запросов/мин (спот) |
| Order rate | 100 ордеров/мин (спот) |
| WebSocket | 100 подписок |

**Безопасный максимум: 8-10 сеток на один API-ключ.**

## PostgreSQL — рост данных

| Таблица | Рост | За месяц (10 сеток) |
|---------|------|---------------------|
| `grid_orders` | ~100 записей/сетка | ~1000 записей |
| `trade_events` | ~50-200/день на сетку | ~15,000-60,000 записей |
| `bot_logs` | ~100-500/день | ~3,000-15,000 записей |
| `audit_log` | ~10-50/день | ~300-1500 записей |

**Диск**: ~1 GB за первый год при 10 активных сетках. 20 GB SSD хватит на 10+ лет.

**Рекомендация**: настройте автоочистку старых логов (например, удаление bot_logs старше 90 дней).

### На что обращать внимание

1. **Локация** — чем ближе к серверам биржи, тем быстрее исполнение:
   - Binance: серверы в Токио, Сингапуре, Франкфурте
   - Bybit: серверы в Сингапуре, Гонконге
   - **Рекомендация**: Франкфурт (Hetzner) для Binance, Сингапур для Bybit

2. **SSD** — обязателен для PostgreSQL. HDD недопустим.

3. **Стабильная сеть** — проверить пинг до API биржи:
   ```bash
   ping api.binance.com
   ping api.bybit.com
   ```

4. **DDoS-защита** — базовая защита у большинства провайдеров бесплатна.

## Масштабирование

### Горизонтальное (несколько серверов)

Если нужно > 15 сеток:
1. **Отдельный сервер для БД** — вынести PostgreSQL + Redis на выделенный сервер
2. **Несколько воркеров** — каждый со своим набором сеток и API-ключей
3. **Балансировщик** — nginx перед несколькими backend-инстансами

### Вертикальное (увеличить мощность)

| Сценарий | CPU | RAM |
|----------|-----|-----|
| 1-5 сеток | 2 vCPU | 4 GB |
| 5-10 сеток | 4 vCPU | 8 GB |
| 10-20 сеток | 4-8 vCPU | 8-16 GB |
| 20+ сеток | 8+ vCPU | 16+ GB |

## Мониторинг ресурсов

### Команды для проверки

```bash
# Потребление контейнеров:
docker stats --no-stream

# Место на диске:
docker system df

# Размер БД:
docker compose exec postgres psql -U moneybot -c "SELECT pg_size_pretty(pg_database_size('moneybot'));"

# Размер таблиц:
docker compose exec postgres psql -U moneybot -c "
SELECT relname, pg_size_pretty(pg_total_relation_size(relid))
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC;"
```

### Когда пора увеличивать ресурсы

- CPU worker постоянно > 80%
- RAM PostgreSQL > 80% лимита
- Дисковое пространство < 20% свободно
- Тики сеток пропускаются (видно в логах)
- Binance/Bybit rate limit ошибки в логах
