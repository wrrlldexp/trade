## 1. Общее описание

MoneyBot v2 — бот для автоматической сеточной торговли криптовалютами на биржах **Binance** и **Bybit**.

Принцип работы: бот раскладывает сетку из ордеров на покупку и продажу вокруг текущей цены. Когда цена падает — бот покупает. Когда растёт — продаёт. Разница между покупкой и продажей — ваша прибыль.

### Что умеет бот:
- 6 торговых стратегий (от простой до адаптивной с капитализацией)
- Работа на Binance и Bybit (спот)
- Paper-режим для безопасного тестирования без реальных денег
- Дашборд с графиками и статистикой в реальном времени
- Многопользовательский режим с ролями
- 2FA-авторизация

## 2. Быстрый старт

### Шаг 1: Вход в систему
Откройте http://localhost:5173 и войдите с учётными данными, которые вам предоставил администратор.

### Шаг 2: Добавьте торговый аккаунт
Перейдите в **Аккаунты** → **Добавить аккаунт**:
- Выберите биржу (Binance или Bybit)
- Вставьте API-ключ и секрет
- Для начала — включите **Тестнет**

### Шаг 3: Создайте сетку
Перейдите в **Сетки** → **Создать сетку**:
- Выберите аккаунт и торговую пару (например BTC/USDT)
- Выберите стратегию (начните с **Простая**)
- Укажите параметры (размер лота, шаг сетки, шаг прибыли)
- Режим: **Paper** для тестов, **Live** для реальной торговли

### Шаг 4: Запустите
Нажмите **Запустить** — бот начнёт работать.

## 4. Торговые аккаунты

### Как получить API-ключи

#### Binance:
1. Войдите на binance.com
2. Профиль → **API Management** → **Create API**
3. Тип: **System generated**
4. Включите: **Enable Spot & Margin Trading**
5. Рекомендуется: задайте **IP Restrictions** (белый список IP вашего сервера)
6. Скопируйте **API Key** и **Secret Key**

> **Важно**: секрет показывается только один раз при создании. Сохраните его.

#### Bybit:
1. Войдите на bybit.com
2. Профиль → **API** → **Create New Key**
3. Тип: **System-generated API Keys**
4. Разрешения: **Trade** (Spot)
5. Скопируйте **API Key** и **API Secret**

### Тестнет (тестовая сеть)
Для безопасного тестирования без риска потерь:
- **Binance Testnet**: зарегистрируйтесь на testnet.binance.vision — там свои ключи
- **Bybit Testnet**: зарегистрируйтесь на testnet.bybit.com — там свои ключи

При добавлении аккаунта включите флаг **Тестнет**.

### Добавление аккаунта в MoneyBot

Перейдите в **Аккаунты** → кнопка **Добавить**:

| Поле | Что указать | Пример |
|------|-------------|--------|
| Название | Любое понятное имя | "Binance основной" |
| Биржа | binance или bybit | binance |
| API Key | Ключ с биржи | xK3jF8... |
| API Secret | Секрет с биржи | s9Kp2m... |
| Тестнет | Вкл — для тестовой сети | ✓ |

> **Безопасность**: Никогда не давайте API-ключам право на **вывод средств** (Withdraw). Для бота нужны только права на торговлю.

## 5. Торговые сетки

### Создание сетки

Перейдите в **Сетки** → **Создать сетку**.

Обязательные поля:
- **Аккаунт** — привязанный биржевой аккаунт
- **Название** — любое имя ("BTC Grid", "ETH Swing" и т.д.)
- **Пара** — торговая пара (BTC/USDT, ETH/USDT и т.д.)
- **Стратегия** — одна из 6 (описание ниже)
- **Режим** — Paper (симуляция) или Live (реальные ордера)
- **Размер лота** — объём одного ордера в базовой валюте
- **Шаг прибыли** — минимальная прибыль с одной сделки
- **Шаг сетки** — расстояние между уровнями
- **Уровней выше** — количество sell-уровней выше текущей цены
- **Уровней ниже** — количество buy-уровней ниже текущей цены
- **Таймаут перестройки** — секунд до пересборки сетки при выходе цены за границы

### Управление сеткой

| Действие | Описание |
|----------|----------|
| **Запустить** | Бот раскладывает ордера и начинает торговлю |
| **Остановить** | Все открытые ордера отменяются, торговля прекращается |
| **Удалить** | Сетка удаляется (только если остановлена) |

### Статусы сетки

| Статус | Значение |
|--------|----------|
| `draft` | Создана, но не запущена |
| `running` | Активно торгует |
| `stopped` | Остановлена вручную |
| `error` | Произошла ошибка |

## 6. Стратегии — подробное описание

MoneyBot поддерживает **6 стратегий**. Каждая определяет, как бот считает прибыль и что делает после завершения цикла buy→sell.

---

### Стратегия 1: Простая (`simple`)

**Принцип**: Фиксированный лот. Купили дёшево → продали дороже → повторили.

**Как работает:**
1. Бот ставит buy-ордер по цене `P`
2. Когда цена падает до `P` — ордер исполняется (бот купил)
3. Бот сразу ставит sell-ордер по цене `P + profit_step`
4. Когда цена растёт до `P + profit_step` — ордер исполняется (бот продал)
5. Бот снова ставит buy-ордер по цене `P` → цикл повторяется

**Прибыль за цикл**: `lot_size × profit_step`

**Пример:**
- lot_size = 0.01 BTC, profit_step = 50 USDT
- Купили 0.01 BTC по 67000 → Продали по 67050
- Прибыль: 0.01 × 50 = **0.50 USDT**

**Кому подходит**: Новичкам. Предсказуемая прибыль, понятная логика.

---

### Стратегия 2: Капитализация (`capitalization`)

**Принцип**: Как простая, но прибыль реинвестируется — лот растёт с каждым циклом.

**Как работает:**
1. Бот работает так же, как простая стратегия
2. После завершения цикла: `новый_лот = старый_лот + (прибыль / цена_покупки)`
3. Каждый следующий цикл торгует чуть большим объёмом

**Пример:**
- Начальный лот: 0.01 BTC, profit_step = 50 USDT, цена покупки = 67000
- После первого цикла: лот = 0.01 + (0.50 / 67000) ≈ 0.01000746 BTC
- После 100 циклов лот будет заметно больше начального

**Кому подходит**: Тем, кто хочет наращивать позицию. Прибыль на прибыль — эффект сложного процента.

---

### Стратегия 3: Реверс (`reverse`)

**Принцип**: Прибыль считается через отношение цен, а не фиксированный шаг.

**Как работает:**
1. Бот покупает и продаёт как обычно
2. Прибыль за цикл: `amount × profit_step / price`
3. После продажи новый buy-ордер ставится с объёмом: `amount + прибыль`

**Формула прибыли**: `lot_size × profit_step ÷ цена_покупки`

**Пример:**
- lot_size = 0.01, profit_step = 50, цена = 67000
- Прибыль: 0.01 × 50 / 67000 ≈ **0.0000075 BTC**

**Кому подходит**: Для рынков с высокой волатильностью. Прибыль адаптируется к цене.

---

### Стратегия 4: Реверс + Капитализация (`reverse_cap`)

**Принцип**: Комбинация реверса и капитализации.

**Как работает:**
1. Прибыль считается по формуле реверса
2. Прибыль реинвестируется в увеличение лота (капитализация)

**Формула**: `новый_лот = старый_лот + (amount × profit_step ÷ price)`

**Кому подходит**: Опытным трейдерам. Максимальная адаптация + рост позиции.

---

### Стратегия 5: Адаптивная (`adaptive`)

**Принцип**: Скользящее окно ордеров, которое следует за ценой. Не все ордера активны одновременно — только те, которые покрыты авансом (prepay).

**Как работает:**
1. Бот строит полную сетку из N уровней
2. Активируется только **подсетка** — диапазон ордеров, покрытых prepay
3. Когда цена уходит за границу подсетки:
   - Подсетка **сдвигается** за ценой
   - Новые ордера получают prepay и флаги **reBuy** / **reSell**
   - Старые ордера за пределами подсетки деактивируются (статус `wait`)
4. Сдвиг происходит с интервалом `adaptive_timer_sec`

**Механизм prepay (аванс):**
- `prepay_base` — общий аванс в базовой валюте (BTC), раздаётся sell-ордерам
- `prepay_quote` — общий аванс в котировочной валюте (USDT), раздаётся buy-ордерам
- `prepay_amount` — сколько аванса выделяется на один ордер
- При исполнении ордера объём = `amount + prepay`

**Флаги reBuy / reSell:**
- **reBuy** — ордер был в зоне сдвига вверх, нужна повторная покупка
- **reSell** — ордер был в зоне сдвига вниз, нужна повторная продажа
- Эти флаги снимаются после исполнения

**Пример:**
- Сетка из 20 уровней, но подсетка охватывает только 8
- Цена выросла → подсетка сдвинулась вверх на 2 уровня
- 2 нижних ордера деактивированы, 2 верхних — активированы с reBuy

**Кому подходит**: Для боковых рынков с медленным трендом. Экономит средства — не нужно держать ордера на всех уровнях.

---

### Стратегия 6: Адаптивная + Капитализация (`adaptive_cap`)

**Принцип**: Адаптивная стратегия + реинвестирование прибыли в увеличение лота.

**Как работает:**
1. Работает как адаптивная (подсетка, prepay, сдвиг)
2. После завершения цикла лот увеличивается: `новый_лот = лот + (прибыль / цена)`
3. При сдвиге подсетки капитализация сбрасывается (лот возвращается к базовому)

**Кому подходит**: Для максимальной автоматизации. Подсетка адаптируется к цене, лот растёт с прибылью.

---

### Сравнительная таблица стратегий

| Стратегия | Прибыль за цикл | Лот растёт? | Подсетка? | Сложность |
|-----------|-----------------|-------------|-----------|-----------|
| Простая | `amount × profit_step` | Нет | Нет | ★☆☆ |
| Капитализация | `amount × profit_step` | Да | Нет | ★★☆ |
| Реверс | `amount × profit_step / price` | Нет | Нет | ★★☆ |
| Реверс+Кап | `amount × profit_step / price` | Да | Нет | ★★★ |
| Адаптивная | `amount × profit_step` | Нет | Да | ★★★ |
| Адаптивная+Кап | `amount × profit_step` | Да | Да | ★★★ |

## 7. Параметры сетки

### Основные параметры

| Параметр | Описание | Влияние | Рекомендация |
|----------|----------|---------|-------------|
| **lot_size** | Объём одного ордера в базовой валюте | Больше лот = больше прибыль, но и больше риск | Начните с минимального: 0.001 BTC |
| **profit_step** | Разница между покупкой и продажей | Больше шаг = реже сделки, но больше прибыль за сделку | 30–100 для BTC/USDT |
| **grid_step** | Расстояние между уровнями сетки | Больше шаг = шире сетка, меньше ордеров | 50–200 для BTC/USDT |
| **levels_above** | Количество sell-уровней выше цены | Больше уровней = больше покрытие вверх | 3–10 |
| **levels_below** | Количество buy-уровней ниже цены | Больше уровней = больше покрытие вниз | 3–10 |
| **rebuild_timeout_sec** | Секунд до пересборки при выходе за границы | Меньше таймаут = быстрее адаптация, но больше нагрузка | 300–3600 |

### Как выбрать параметры

**Для BTC/USDT** (цена ~67 000):
```
lot_size: 0.001        # ~67 USDT на ордер
profit_step: 50        # 50 USDT прибыли за цикл
grid_step: 100         # 100 USDT между уровнями
levels_above: 5        # 5 sell-ордеров
levels_below: 5        # 5 buy-ордеров
```

**Для ETH/USDT** (цена ~3 500):
```
lot_size: 0.01         # ~35 USDT на ордер
profit_step: 10        # 10 USDT прибыли за цикл
grid_step: 20          # 20 USDT между уровнями
levels_above: 5
levels_below: 5
```

### Формула: сколько нужно средств

```
Минимум USDT = lot_size × средняя_цена_уровня × levels_below
Минимум Base = lot_size × levels_above
```

Пример для BTC/USDT, lot = 0.001, 5 уровней, grid_step = 100:
- На buy: 0.001 × (66900 + 66800 + 66700 + 66600 + 66500) ≈ **333.5 USDT**
- На sell: 0.001 × 5 = **0.005 BTC**

## 8. Адаптивные стратегии — расширенные параметры

Эти параметры нужны только для стратегий **Адаптивная** и **Адаптивная+Кап**.

| Параметр | Описание | Рекомендация |
|----------|----------|-------------|
| **adaptive_timer_sec** | Интервал проверки сдвига подсетки (секунды) | 10–30 |
| **prepay_base** | Общий аванс в базовой валюте (BTC) для sell-ордеров | Зависит от количества уровней: `prepay_amount × кол-во_sell_уровней` |
| **prepay_quote** | Общий аванс в котировочной валюте (USDT) для buy-ордеров | `prepay_amount × средняя_цена × кол-во_buy_уровней` |
| **prepay_amount** | Аванс на один ордер | 10–50% от lot_size |

### Пример настройки адаптивной стратегии

BTC/USDT, 10 уровней, подсетка из 6:
```
lot_size: 0.001
profit_step: 50
grid_step: 100
levels_above: 5
levels_below: 5
adaptive_timer_sec: 15
prepay_amount: 0.0005          # 50% от лота
prepay_base: 0.0025            # 0.0005 × 5 sell-уровней
prepay_quote: 167.5            # 0.0005 × 67000 × 5 buy-уровней
```

### Как работает подсетка — визуально

```
Цена: 67000 USDT

Уровень 10:  68000  SELL  [wait]       ← за пределами подсетки
Уровень 9:   67800  SELL  [wait]       ← за пределами подсетки
Уровень 8:   67600  SELL  [placed] ⬆   ← верхняя граница подсетки (top_idx)
Уровень 7:   67400  SELL  [placed]
Уровень 6:   67200  SELL  [placed]
--- текущая цена 67000 ---
Уровень 5:   66800  BUY   [placed]
Уровень 4:   66600  BUY   [placed]
Уровень 3:   66400  BUY   [placed] ⬇   ← нижняя граница подсетки (bottom_idx)
Уровень 2:   66200  BUY   [wait]       ← за пределами подсетки
Уровень 1:   66000  BUY   [wait]       ← за пределами подсетки
```

Когда цена вырастет до 67500:
```
Уровень 10:  68000  SELL  [placed] ⬆   ← подсетка сдвинулась вверх
Уровень 9:   67800  SELL  [placed]
Уровень 8:   67600  SELL  [placed]
--- текущая цена 67500 ---
Уровень 7:   67400  BUY   [placed] reBuy
Уровень 6:   67200  BUY   [placed]
Уровень 5:   66800  BUY   [placed] ⬇
Уровень 4:   66600  BUY   [wait]       ← деактивирован
Уровень 3:   66400  BUY   [wait]       ← деактивирован
```

## 10. Управление пользователями

> Доступно только для роли **Суперадмин**.

### Роли

| Роль | Возможности |
|------|------------|
| **Суперадмин** | Полный доступ: пользователи, аудит, сетки, аккаунты |

**Способ 1 — Прямое создание** (кнопка "Создать"):
1. Перейдите в **Пользователи**
2. Нажмите **Создать**
3. Заполните: Email, Имя, Пароль, Роль
4. Аккаунт создаётся мгновенно — пользователь может сразу войти

**Способ 2 — Приглашение по ссылке** (кнопка "Пригласить"):
1. Укажите Email и Роль
2. Скопируйте сгенерированную ссылку
3. Отправьте ссылку пользователю
4. Он перейдёт по ней, задаст имя и пароль
5. Ссылка действует 7 дней

### Управление существующими пользователями
- **Деактивировать** — пользователь не сможет войти
- **Активировать** — восстановить доступ
- **Удалить** — деактивация аккаунта

## 11. Режимы работы: Paper и Live

### Paper (симуляция)
- Ордера **не** отправляются на биржу
- Бот симулирует исполнение по текущей цене
- Безопасно для тестирования стратегий
- Можно использовать с любыми API-ключами (даже невалидными)
- **Рекомендуется** для первого знакомства с ботом

### Live (реальная торговля)
- Ордера отправляются на реальную биржу
- Требуются валидные API-ключи с правом торговли
- Бот автоматически обрабатывает:
  - Rate limits (ограничения биржи на количество запросов)
  - Ошибки сети (переподключение с задержкой)
  - Ошибки биржи (Binance: -1003, -1015, -1021; Bybit: 10006)

### Тестнет
- Биржи предоставляют тестовые сети с виртуальными средствами
- Live-режим + тестнет = реальные ордера на тестовой бирже
- Идеально для проверки перед запуском на реальные деньги

## 14. Управление ботом

### Панель управления на дашборде

На главной странице (Дашборд) для админов и суперадминов отображается **панель управления ботом**:

| Элемент | Описание |
|---------|----------|
| **Индикатор статуса** | Зелёный = воркер онлайн, Красный = оффлайн |
| **Активные сетки** | Количество сеток, которые сейчас торгуют |
| **Остановить всё** | Экстренная остановка всех активных сеток (admin+) |
| **Перезагрузить** | Перезагрузка воркера (только superadmin) |

### Как это работает

- Воркер отправляет **heartbeat** в Redis каждые 10 секунд
- Если heartbeat не обновлялся > 35 секунд — воркер считается оффлайн
- Кнопка **Остановить всё** отправляет команду через Redis pub/sub → воркер отменяет все ордера и останавливает все сетки
- Кнопка **Перезагрузить** отправляет команду рестарта → Docker автоматически перезапустит воркер

### Когда использовать

- **Остановить всё** — при подозрении на неправильное поведение бота, резком движении рынка, или перед обслуживанием
- **Перезагрузить** — если воркер завис или нужно применить обновления кода

## 16. FAQ и устранение проблем

### Бот не торгует, статус "running" но трейдов нет
- Проверьте, что цена находится внутри диапазона сетки
- Для адаптивной: проверьте, что подсетка построена (видно на странице сетки)
- Убедитесь, что достаточно средств на балансе биржи

### Ордера не ставятся, ошибка "insufficient_funds"
- Недостаточно средств на балансе. Проверьте формулу расчёта из раздела 7
- Убедитесь, что средства не заблокированы в других ордерах

### Ошибка "invalid_order"
- Объём ордера меньше минимального на бирже
- Binance: минимум обычно 10 USDT за ордер
- Bybit: проверьте минимальные лоты для конкретной пары

### Ошибка "permanent:-2015" или "permanent:10003"
- Невалидный API-ключ. Проверьте, что ключ активен и не истёк
- Убедитесь, что ключ от нужной сети (mainnet / testnet)

### Как остановить все сетки?
- **Быстрый способ**: На дашборде нажмите кнопку **Остановить всё** — это мгновенно остановит все активные сетки
- **По одной**: Каждую сетку можно остановить индивидуально через кнопку **Стоп**
- При остановке все открытые ордера автоматически отменяются

### Воркер оффлайн, что делать?
- Проверьте статус контейнера: `docker compose ps worker`
- Посмотрите логи: `docker compose logs worker`
- Перезапустите: `docker compose restart worker`
- Или через UI: кнопка **Перезагрузить** на дашборде (только superadmin)

### Где посмотреть ошибки?
- Раздел **Логи** (`/logs`) — фильтр по уровню `error` или `critical`
- Кликните на ошибку, чтобы увидеть traceback и место в коде
- Также доступно через Docker: `docker compose logs backend` / `docker compose logs worker`

### Как пересобрать сетку?
- Для стратегий 1–4: бот автоматически пересоберёт сетку, когда цена выйдет за границы и пройдёт `rebuild_timeout_sec` секунд
- Для адаптивных (5–6): подсетка сдвигается автоматически каждые `adaptive_timer_sec` секунд

### Сколько сеток можно запустить одновременно?
- Технически — неограниченно
- Практически: зависит от rate limits биржи. Каждая сетка потребляет ~8 weight на тик (Binance лимит 1200/мин). Безопасно: до 10–15 сеток на один API-ключ

### Данные на дашборде обновляются с задержкой
- Дашборд обновляется каждые 30 секунд
- Для мгновенного обновления — обновите страницу (F5)
